# 도입부 화면 수정 가이드 (서울시민대학용)

이 저장소의 `opening/index.html` 파일이 특강 도입부에 프로젝터로 띄우는 화면입니다.

- 실제 송출 화면 미리보기: <https://portfolio-school-1st-gen.github.io/final_class/opening/>

이 문서는 **개발 도구 없이, 웹브라우저와 ChatGPT만으로** 문구·디자인을 직접 수정해서
기획팀에 반영 요청(Pull Request)까지 보내는 방법을 안내합니다.
낯선 용어가 나와도 괜찮습니다 — 순서대로 따라 하시면 됩니다.

> 중간에 막히면 언제든 기획팀에 연락 주세요.
> 수정한 파일을 메일로 보내주셔도 저희가 대신 올려드립니다.

## 준비물

- 이메일 주소 (GitHub 가입용)
- ChatGPT (<https://chatgpt.com> — 무료 계정으로 충분합니다)
- 디자인 기준으로 삼을 **포스터 이미지 파일**

## 1단계 — GitHub 계정 만들기 (약 5분)

GitHub는 이 화면 파일이 보관된 곳입니다. 수정 제안을 보내려면 계정이 필요합니다.

1. <https://github.com> 접속 → 오른쪽 위 **Sign up** 클릭
2. 이메일 주소 → 비밀번호 → 사용자 이름(영문) 순서로 입력
3. 이메일로 온 인증 코드를 입력하면 가입 완료
4. 요금제를 묻는 화면이 나오면 **Free(무료)** 를 선택

## 2단계 — 파일 내려받고 현재 화면 확인하기

1. 이 저장소 첫 화면(<https://github.com/portfolio-school-1st-gen/final_class>)에서
   초록색 **Code** 버튼 → **Download ZIP** 클릭
2. 내려받은 ZIP 파일 압축 풀기
3. 그 안의 **`opening` 폴더 → `index.html`** — 이것이 수정 대상 파일입니다
   - 같은 폴더의 `assets`에는 QR코드·연사 사진이 들어 있습니다 (건드리지 않으셔도 됩니다)
4. `index.html`을 더블클릭하면 브라우저에서 현재 화면이 열립니다
   - **← → 방향키**로 슬라이드가 넘어갑니다 (총 6장)

## 3단계 — ChatGPT로 수정하기

1. <https://chatgpt.com> 에서 새 대화 시작
2. 입력창의 **클립(+) 아이콘**으로 `index.html` 파일과 **포스터 이미지**를 함께 첨부
3. 아래 요청문을 복사해서 붙여넣되, `[ ]` 부분만 원하는 내용으로 바꿔서 보내기

```text
첨부한 index.html은 행사(7/24 멘토특강) 도입부에 프로젝터로 띄우는 웹 슬라이드입니다.
함께 첨부한 포스터 이미지의 디자인 톤(색감·분위기·서체 느낌)에 맞춰 슬라이드를 수정해 주세요.

[여기에 원하는 수정 내용을 자유롭게 적으세요.
 예) 전체 배경을 포스터처럼 ○○색 계열로 바꿔줘
 예) 2번 슬라이드의 "..." 문구를 "..."로 바꿔줘]

반드시 지켜야 할 조건:
1. 파일 아래쪽 <script> 부분(키보드 조작 코드)은 수정하지 말 것
2. 이미지 경로 assets/qr.png 와 assets/speaker.jpg 는 그대로 유지할 것
3. 인터넷에서 불러오는 외부 리소스(웹폰트 CDN, 외부 이미지 링크 등)는 추가하지 말 것
4. 1280×720 크기의 #stage 구조와 슬라이드 6장 구성은 유지할 것
5. 다 되면 수정된 파일 "전체"를 처음부터 끝까지 하나의 코드 블록으로 출력할 것
```

4. ChatGPT가 수정된 파일 전체를 코드 블록으로 출력하면,
   코드 블록 오른쪽 위 **Copy(복사)** 버튼으로 전체 복사
5. 압축 풀어둔 폴더의 `opening/index.html`을 **메모장**으로 열기
   (파일 우클릭 → 연결 프로그램 → 메모장) → 전체 선택(`Ctrl+A`) → 붙여넣기(`Ctrl+V`) → 저장(`Ctrl+S`)
6. `index.html`을 다시 더블클릭해서 결과 확인 — 마음에 들 때까지 3~6을 반복

> **답변이 중간에 잘렸다면?** 붙여넣은 파일의 맨 끝이 `</html>`로 끝나지 않으면 잘린 것입니다.
> ChatGPT에게 "파일 전체를 처음부터 끝까지 잘리지 않게 다시 출력해줘"라고 요청하세요.

## 4단계 — 수정 내용을 GitHub로 보내기 (Pull Request)

1. 저장소에서 `opening/index.html` 파일을 클릭해서 열기
   (바로가기: <https://github.com/portfolio-school-1st-gen/final_class/blob/main/opening/index.html>)
2. 오른쪽 위 **연필 아이콘(Edit this file)** 클릭
3. "Fork this repository" 안내가 나오면 초록 버튼 클릭
   - 내 계정에 복사본을 만든다는 뜻입니다. **원본은 바뀌지 않으니 안심하세요**
4. 편집 화면에서 전체 선택(`Ctrl+A`) → 삭제 → 3단계에서 완성한 파일 내용을 통째로 붙여넣기
5. 오른쪽 위 초록색 **Commit changes...** 클릭
6. 설명 칸에 무엇을 바꿨는지 한 줄 적기 (예: "포스터 톤에 맞춰 배경색 변경") → **Propose changes**
7. 다음 화면에서 **Create pull request** 초록 버튼 → 한 번 더 **Create pull request**
8. 끝입니다! 기획팀이 내용을 확인한 뒤 실제 화면에 반영하고, 반영되면
   <https://portfolio-school-1st-gen.github.io/final_class/opening/> 에서 확인하실 수 있습니다

## 자주 묻는 것

- **실수하면 어떡하죠?** — Pull Request는 "이렇게 바꾸면 어떨까요?"라는 *제안*입니다.
  기획팀이 확인한 뒤에만 실제 화면에 반영되므로, 어떤 실수도 원본을 망가뜨리지 않습니다.
- **새 이미지를 넣고 싶어요** — 이미지 파일을 기획팀에 보내주세요. 저장소에 올린 뒤 사용법을 알려드립니다.
- **여러 번 나눠서 수정해도 되나요?** — 네, 몇 번이든 괜찮습니다.
- **주의**: 이 저장소는 누구나 볼 수 있는 공개 저장소입니다.
  개인 연락처, 내부 문서 내용 등은 올리지 말아 주세요.

## (기획팀 검수용) 반영 전 확인 항목

아래 조건은 ChatGPT 요청문에 이미 포함되어 있고, PR 검토 시 기획팀이 다시 확인합니다.

- [ ] `<script>` 키보드 조작부 원형 유지 (방향키·숫자·`B`/`F`/`H`, `e.code` 기반 한/영 무관 판정)
- [ ] `assets/qr.png`(질문 수집 QR)·`assets/speaker.jpg` 경로와 노출 유지
- [ ] 외부 리소스(CDN 웹폰트·외부 이미지) 추가 없음 — 행사장 오프라인 구동 전제
- [ ] 1280×720 `#stage` + 자동 스케일(fit) 구조 유지, `@media print`(PDF 백업) 유지
- [ ] 헤드리스 캡처로 전 슬라이드 렌더 확인 후 머지
